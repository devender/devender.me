* Probably the best session I have attended so far is the **[JavaPossie](http://www.javaposse.com/)** BOF, for those of you who don’t know what the **JavaPossie** is it is a podcast which is centered around java and I highly recommend it. I was very happy to finally see the show done in person.
* The day started off with a general session from **Intel**, like AMD, Intel too is working with Sun to optimize the **JVM** for there platform so with newer versions of the JVM it should work faster straight out of the box, without you having to do anything !! hey that’s the sort of **performance** improvement that I like 😉 they claim a performance improvement of about 60% on existing hardware just using a newer VM.
* Check out [moblin.org](http://moblin.org/) , it is an Intel sponsored open source project. Also the [**atom processor**](http://www.intel.com/technology/atom/index.htm), which is Intel’s smallest chip.
* The Garbage-First Garbage Collector or G1 for short is the next-generation low-pause garbage collector that will be included in the Java HotSpot virtual machine. G1 will be the long-term replacement for the Concurrent Mark-Sweep (or CMS) garbage collector, Sun’s current low-pause garbage collector ( [Presentation](https://devender.wordpress.com/wp-content/uploads/2008/05/garbage.pdf) )
* [Scalable Non Blocking Data Structure](http://sourceforge.net/projects/high-scale-lib/), was presented by [Dr. Cliff](http://blogs.azulsystems.com/cliff/) and all the code can be found on [Source Forge](http://sourceforge.net/projects/high-scale-lib/) the code is based on **Lock Free algorithms**. ([Presentation](https://devender.wordpress.com/wp-content/uploads/2008/05/nonblocking.pdf))
* Performance talk, basically any component in your system can become a bottle neck from Disk IO, DB etc. best approach it to address one issue at a time.
* [Pluse Live Scribe](http://www.livescribe.com/), was just too cool, it is a pen that runs Java.
* [Call of the Kings](http://www.callofthekings.com/main.php), game done in Java.
* [JMonkey](http://www.jmonkeyengine.com/) a J2ee for games.
* [Esper](http://esper.codehaus.org/index.html) for complex event processing.
